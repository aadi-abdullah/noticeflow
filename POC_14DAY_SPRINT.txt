                        WHAT SUCCESS LOOKS LIKE
===============================================================================

CA uses your tool and says:

  "Hmm, this is mostly right. I'd still verify, but it saves me reading
   through the whole notice manually. Yeah, I'd use this on my next notice."

NOT:
  "Wow, this is amazing AI!" (too much hype)
  "This is completely wrong" (core idea broken)

YOU WANT: Quiet utility, not excitement.

===============================================================================
                        WHAT FAILURE LOOKS LIKE
===============================================================================

CA uses your tool and says:

  "The deadline is wrong."
  "This checklist is too generic."
  "I don't trust this at all."
  "Why would I use this?"

→ Do NOT add features
→ Fix the core issue or kill the project

===============================================================================
                         EMERGENCY PROTOCOL
===============================================================================

IF process_notice.py crashes on a notice:
  □ Capture the error message
  □ Log in failure_log.csv
  □ Test with simpler notice (digital PDF)
  □ If still broken: debug OCR pipeline
  □ If LLM error: check API key + quota
  □ If prompt error: simplify prompt

IF CA testing feedback is very negative:
  □ Do NOT panic
  □ Ask: "What specifically is wrong?"
  □ Is it fixable with prompt iteration? (3-5 days)
  □ Is it a core concept problem? (kill or pivot)

IF response time is >30 seconds:
  □ Profile where time is spent:
    - OCR? → Switch to GPT-4o mini or simpler model
    - LLM? → Try model caching or prompt compression
    - Network? → Not much you can do in PoC

IF accuracy is <80%:
  □ Don't blame the model
  □ Blame the prompt
  □ Add more examples + clearer instructions
  □ Retest
